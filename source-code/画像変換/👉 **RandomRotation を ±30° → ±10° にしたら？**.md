これも本質的に**「データ拡張をどこまで信じるか」**の調整です。

結論から言うと、**医療画像では ±10° に下げる判断はかなり妥当**です。

---

## **結論を一言で**

- **±30°**：汎化は強いが、医療画像では「やりすぎ」になりやすい
    
- **±10°**：現実的・安定・医療向き（おすすめ）
    

---

## **何がどう変わる？**

  

### **① モデルが学ぶ「回転不変性」の強さ**

- **±30°**
    
    - 大きく傾いた画像も「同じクラス」と学ぶ
        
    - 回転にかなり強くなる
        
    
- **±10°**
    
    - 軽微な傾きだけ許容
        
    - 現実の撮影ブレに近い
        
    

  

👉 **どこまで回転しても意味が変わらないか？**

それをモデルに教えている、と思ってください。

---

### **② 医療画像との相性**

  

乳がん・病理画像では：

- スライドの上下方向
    
- 組織構造の配向
    
- 染色の流れ
    

  

などが **完全に回転不変とは限らない**。

  

👉 **±30° は「存在しないデータ」を作りすぎる可能性**

👉 **±10° は「現実的な揺らぎ」だけを表現**

---

## **学習挙動の違い（経験則）**

| **観点**     | **±30°** | **±10°** |
| ---------- | -------- | -------- |
| train loss | 下がりにくい   | 下がりやすい   |
| val acc    | 上がるが不安定  | 安定しやすい   |
| seed依存     | 大きい      | 小さい      |
| 収束速度       | 遅い       | 速い       |
## **他の拡張との関係（重要）**

  

あなたの構成は：

- RandomCrop
    
- RandomHorizontalFlip
    
- RandomRotation
    
- RandomAffine（translate / scale / shear）
    

  

ここで **Rotation だけ強い**と、

  

> 拡張が「回転寄り」に偏る

  

という問題が出ます。

  

👉 Rotation を **±10°** に下げると

👉 translate / scale / shear とバランスが取れる
