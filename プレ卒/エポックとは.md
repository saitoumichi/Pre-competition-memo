epoch（エポック）とは：
　Epoch＝学習データを一回全て使い切ること
　学習用データが1000件あるとします
　1epochとは
　・1−1000番目までのデータを
　・全て使って
　・重みを更新し終えた状態
　＝＞これが1epoch
　必要な理由として：
　ニューラルネットワークは
　・最初は重みがランダム
　・1回見ただけでは何もわからない
　だから
　・同じデータを
　・何周も
　・少しずつ理解していく
　＝＞その何周目かを表す単位がEpoch
　Epoch・Batch・Iterationの関係（超重要）
　　・データ数：1000
　　・batch size：100
　　　
| 用語        | 意味           | 回数          |
| --------- | ------------ | ----------- |
| Batch     | 一度に処理するデータの塊 | 100件        |
| Iteration | 1batch処理     | 10回 / epoch |
| Epoch     | 前データを一周      | 1回          |

  1 epoch
 　　├─ iteration 1（1〜100）
 　　├─ iteration 2（101〜200）
　　 ...
 　　└─ iteration 10（901〜1000）
　
　Epochを増やすと何が起こるのか
　良い場合：
　・重みが洗練される
　・精度が上がる
　悪い場合：
　・覚えすぎる（過学習）
　・検証精度が下がる
　
| Epoch | 学習精度 | 検証精度     |
| ----- | ---- | -------- |
| 5     | 70％  | 68％      |
| 20    | 90％  | 88％      |
| 50    | 99％  | 85％＜＝過学習 |
＝＞検証精度が下がり始めたら止める
　　どう決めるのか：
　　よくある方法として
　　・固定値（例：50、100）
　　・検証精度で自動停止（推奨）
　　目安：

| データ量 | Epoch |
| ---- | ----- |
| 少ない  | 多め    |
| 多い   | 少なめ   |
