
おすすめライブラリ（用途別）

> **画像分類・特徴抽出 → timm**

> **NLP / Transformer → Hugging Face**

> **研究・自作モデル → PyTorch Lightning**

> **高速実装・プロダクション → TorchVision / ONNX**

> **評価・実験管理 → scikit-learn / wandb**

## **① Hugging Face 🤗 Transformers（超重要）**
### **何ができる？**

- BERT / GPT / T5 / ViT / CLIP など
- NLP・Vision・Audio・マルチモーダル

`from transformers import AutoModel`
`model = AutoModel.from_pretrained("bert-base-uncased")`

### **強い理由**
- 論文モデルが即使える
- 学習済み重みが豊富
- APIが統一されている
👉 **Transformer系なら事実上の標準**

## **② torchvision（PyTorch公式）**
### **何ができる？**
- ResNet / MobileNet など基本CNN
- Dataset / Transform
### **特徴**
- 安定・軽量
- 依存関係が少ない
👉　**「最小構成で学びたい」ならこれ**


## **③ PyTorch Lightning（学習管理の神）**
### **何ができる？**
- 学習ループを自動化
- 再現性・可読性UP

`class LitModel(pl.LightningModule):`
    `...`

### **強い理由**
- train / val / test を強制分離
- 実験が事故りにくい
👉　**研究・レポート向き**

## **④ scikit-learn（評価・前処理の王）**
### **何ができる？**
- 交差検証
- 評価指標
- 前処理
```
from sklearn.model_selection import KFold
```
＝＞正しく評価するために必須

## **⑤ Detectron2 / MMDetection（検出・セグメンテーション）**
### **何ができる？**
- 物体検出
- セグメンテーション
👉　**画像分類以外ならここ**

## **⑥ ONNX / TensorRT（本番・高速化）**
### **何ができる？**
- 推論高速化
- デプロイ
👉　**研究→実サービスの橋渡し**

## **⑦ Weights & Biases（実験ログ管理）**
### **何ができる？**
- 学習曲線可視化
- ハイパーパラメータ管理
👉**「何を試したか分からなくなる」問題を解決**
